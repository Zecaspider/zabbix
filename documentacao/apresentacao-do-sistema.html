04Arquitectura — 3 camadas, cada uma com a sua fonte de verdade

Regra estrutural: o Grafana não duplica alertas. O alarme vive no Zabbix; o Grafana mostra, navega e correlaciona. Isto evita dois sistemas de alerta a divergir.

05Como foi montado

1 · Tudo é código versionado

Cada painel é um ficheiro no repositório; nada se edita "à mão" no Grafana. O fluxo é sempre editar local → validar → publicar → testar → commit. Qualquer dashboard pode ser reconstruído, auditado ou revertido.

2 · Auditoria antes de desenhar

Nenhum dashboard nasceu de suposições: cada segmento passou por checklist contra os dados reais do Zabbix, porque tags e nomes herdados provaram estar errados várias vezes.

3 · Construção incremental

Cada painel é escrito, publicado e testado individualmente antes do seguinte — nunca "big bang".

4 · Runtime partilhado

Um painel utilitário por dashboard fornece tema, cores e thresholds comuns — o N1 nunca contradiz o N2/N3.

5 · Segurança por contrato

Leituras livres; qualquer escrita (Zabbix, Grafana, VMs) exige confirmação humana explícita, caso a caso.

+ O trabalho invisível

Saneamento da base de monitorização: agentes recuperados, 23 hosts duplicados removidos, reclassificação host a host das BDs, tags coerentes nos 41 sintéticos, reconciliação dos 50 sistemas de negócio. Um dashboard só vale o que valem os dados por baixo.

06Operação NOC — rotina de turno

Acesso: http://10.10.126.22:3000 → pasta 00 · Visão Geral → N1 · Visão Geral — Estado global (wallboard, refresh ao minuto).

  1. Wallboard N1 no ecrã grande

    8 cards, um por domínio, com contagem de CRÍTICOS e AVISOS. Verde = seguir; vermelho = triagem.

  2. Triagem por drill-down

    Clicar "Ver N2 →" no card afectado e descer nível a nível. Ex.: agência sem comunicação → N2 Rede → N3 Agências (geomap) → N4 Agência → N5 Interfaces do router. Sistema de negócio em baixo → N2 APIs → N3 App → N4 Sistema (sintético + VMs que o suportam).

  3. Confirmar no Zabbix quando necessário

    O alarme oficial (trigger, duração, ack) vive no Zabbix; o Grafana diz onde olhar.

  4. Relatórios

    Dashboard R0 · Relatórios exporta Excel/PDF por domínio; o R1 · Relatório Diário cobre os 50 sistemas de negócio.

Convenções de leitura

Verde OK · amarelo aviso · vermelho crítico · cinzento sem dados. Badge "dados desactualizados" = a fonte parou de reportar (sinal real, não erro do ecrã). Painéis compostos demoram 10–20 s no primeiro load.

Incidente conhecido

Dashboards sem dados após corte de energia no servidor Grafana → seguir o runbook incidente-plugin-zabbix-grafana-20260710.md (3 passos, sem SSH). Não reinstalar nada às cegas.

O operador NÃO faz

  • Editar dashboards directamente no Grafana
  • Activar templates/discovery/notificações no Zabbix (risco de flood de tickets — incidente GLPI documentado)
  • Apagar ou mover dashboards entre pastas

07Manutenção & roadmap

Qualquer alteração segue o mesmo ciclo: editar .js local → validar sintaxe → push ao Grafana → testar no browser → git commit. O Grafana é o ambiente de teste; o Git é o repositório. Documentação actualizada logo após cada aprovação.
